**Операции Web Server API**

3.1. В запрос *{{WebServer}}/content/list* добавить два query-параметра: sortType и *sortDirection*.   
Поле sortType может принимать два значения:

- Если “sortType”: “rating”, внутри массива сортировка элементов должна выполняться по полю *rating (зрительский рейтинг)*;  
- Если “sortType”: “criticsRating”, *внутри массива сортировка элементов должна выполняться по полю criticsRating (рейтинг кинокритиков)*.

Поле direction может принимать два значения:

- Если “*sortDirection*”: “desc”, внутри массива элементы должны сортироваться по параметру sortType по убыванию;  
- *Если “sortDirection”: “asc”, внутри массива элементы должны сортироваться по параметру sortType по возрастанию;*

*Ограничения сортировки по полям criticsRating (рейтинг кинокритиков) и rating (зрительский рейтинг):*

- *Если “sortType” \== “rating” и значение поля “rating” \== 0, то такие элементы всегда должны выводиться в конце списка*.   
- *Если “sortType” \== “criticsRating” и значение поля “criticsRating” \== 0, то такие элементы всегда должны выводиться в конце списка.*
